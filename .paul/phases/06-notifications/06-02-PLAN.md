<task type="auto">
  <name>Task 2: Build NotificationSetupScreen with permission handling</name>
  <files>app/(tabs)/settings.tsx</files>
  <action>
    Replace the settings.tsx stub entirely with a NotificationSetupScreen:

    1. Imports:
       - React, useState, useCallback, useRef from react
       - View, Text, Switch, Pressable, StyleSheet, Linking from react-native
       - { useFocusEffect } from expo-router <!-- audit-added: re-check permission on screen focus -->
       - * as Notifications from expo-notifications
       - Screen, Button from @components/index
       - useNotificationPrefsStore from @store/notificationPrefsStore
       - scheduleSlotNotifications from @lib/notifications
       - NotificationSlot from @models/index
       - Design tokens from @constants/theme

    2. Component structure:
       a. On screen focus (useFocusEffect from expo-router): <!-- audit-added: must re-check permission when returning from OS Settings -->
          - Check current permission status via Notifications.getPermissionsAsync()
          - Store result in local state: permissionStatus (granted/denied/undetermined)
          - If permission state changed from stored value, sync store: store.setPermissionGranted(result.granted) <!-- audit-added: keeps store in sync with OS state -->
          - Use useFocusEffect (NOT useEffect) — screen doesn't remount when returning from Linking.openSettings(), so useEffect([]) would show stale permission state
          - Read stored prefs from useNotificationPrefsStore hook

       b. Permission-denied banner (shown when permissionStatus !== 'granted'):
          - Yellow/warning-colored banner at top of screen
          - Text: "Notifications are disabled. Enable them in Settings to receive reminders."
          - Pressable: tapping calls Linking.openSettings()
          - accessibilityRole="button", accessibilityHint="Opens device settings to enable notifications"

       c. Slot toggles section:
          - Section header: "Reminder times"
          - 4 rows, one per slot (morning/afternoon/evening/pre-sleep)
          - Each row: label + hint text on left, Switch on right
          - Switch value matches selectedSlots.includes(slot)
          - On toggle: update store + reschedule
          - Disabled if permission not granted (visual + accessibilityState)

       d. Streak nudge toggle section:
          - Section header: "Streak reminder"
          - Single row: "Evening nudge" / "Reminds you at 8pm if you haven't logged today"
          - Switch bound to streakNudgeEnabled
          - On toggle: update store + reschedule
          - Disabled if permission not granted

       e. Reschedule helper (useCallback):
          - Read current slots and streakNudge from store getState()
          - Call scheduleSlotNotifications(slots, streakNudge)
          - Wrap in try-catch with __DEV__ console.error
          - MUST debounce with ~300ms delay — rapid switch toggling causes concurrent cancel-all-then-schedule flows that interleave, producing duplicate notifications. Use a useRef timer: clear previous timeout, set new one. <!-- audit-added: race condition on rapid toggle -->

    3. Styling:
       - Use Screen scrollable={true}
       - Use design tokens (colors, typography, spacing, radius)
       - Banner: backgroundColor colors.errorLight with colors.text text (WCAG compliant per theme annotation)
       - Switch: trackColor uses colors.primary (on) and colors.border (off)
       - Rows: paddingVertical spacing.md, borderBottomWidth 1, borderBottomColor colors.borderLight
       - Section headers: typography.sizes.sm, fontWeight semibold, color textSecondary, uppercase

    4. Accessibility:
       - Switch components: accessibilityLabel describing the slot and current state
       - Permission banner: accessibilityRole="button"
       - Section headers: accessibilityRole="header"
       - Disabled switches: accessibilityState={{ disabled: true }}

    Avoid: Do NOT read entries or compute streaks on this screen — that's Phase 7 territory.
    Avoid: Do NOT add account management, theme toggle, or app version — those are Phase 9.
  </action>
