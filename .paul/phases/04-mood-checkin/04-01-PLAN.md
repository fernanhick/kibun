<objective>
## Goal
Build the complete mood check-in flow: mood selection grid, confirmation screen with optional note and Shiba reaction, and local persistence with Supabase sync for registered users. After this plan, a user can tap "Log mood" on HomeScreen, pick from 14 mood bubbles, add an optional note, submit, and see the entry saved.

## Purpose
The mood check-in is the core user action in kibun. Without it, the app has onboarding and auth but no primary function. This plan delivers the testable MVP loop: check in → save → return to home.

## Output
- `app/check-in.tsx` — MoodSelectionScreen: 14 mood bubbles in grouped grid
- `app/mood-confirm.tsx` — MoodConfirmScreen: selected mood display, optional note, Shiba reaction, submit
- `src/store/moodEntryStore.ts` — Zustand store with AsyncStorage persistence + Supabase insert for registered users
- `src/lib/checkInSlot.ts` — Time-of-day slot detection (morning/afternoon/night/pre_sleep)
- `src/types/index.ts` — MoodEntry type added
- `app/(tabs)/index.tsx` — "Log mood" CTA on HomeScreen
- `app/_layout.tsx` — check-in and mood-confirm Stack.Screens registered
</objective>

      id: string;           // uuid — generated client-side

    ── 2. checkInSlot helper (src/lib/checkInSlot.ts) ──────────────────────────

    Pure function that maps current hour to MoodSlot:

    ```ts
    import { MoodSlot } from '@models/index';

    export function getCheckInSlot(date: Date = new Date()): MoodSlot {
      const hour = date.getHours();
      if (hour >= 5 && hour < 12) return 'morning';
      if (hour >= 12 && hour < 17) return 'afternoon';
      if (hour >= 17 && hour < 21) return 'night';
      return 'pre_sleep';
    }
    ```

    ── 3. moodEntryStore (src/store/moodEntryStore.ts) ─────────────────────────

    Zustand store with AsyncStorage persistence. Follow the same pattern as sessionStore.

    State shape:
    ```ts
    interface MoodEntryState {
      entries: MoodEntry[];
      addEntry: (entry: MoodEntry) => void;
      getToday: () => MoodEntry[];
    }
    ```

    - `entries` array persisted to AsyncStorage (key: 'kibun-mood-entries')
    - `addEntry` prepends new entry to array AND attempts Supabase insert if user is registered
    - `getToday` filters entries by today's date (compare loggedAt date portion)

    For Supabase sync inside addEntry:
    - Read session from useSessionStore.getState() (non-reactive, OK for fire-and-forget)
    - If session?.authStatus === 'registered', fire Supabase insert with EXPLICIT field mapping:
      <!-- audit-added M1: mood_entries table schema differs from MoodEntry interface -->
      ```ts
      supabase.from('mood_entries').insert({
        user_id: session.userId,
        mood: entry.moodId,
        mood_color: MOOD_MAP[entry.moodId].bubbleColor,
        note: entry.note,
        check_in_slot: entry.slot,
        logged_at: entry.loggedAt,
        // Do NOT send entry.id — Supabase auto-generates uuid PK
      })
      ```
      Import MOOD_MAP from '@constants/moods' in the store file.
    - Wrap in try/catch — Supabase failure must NOT prevent local save
    - Log error in __DEV__ only

    Generate entry ID with: `Date.now().toString(36) + Math.random().toString(36).substring(2, 9)`
    (Lightweight, unique enough for local AsyncStorage indexing. NOT sent to Supabase.)

    ── 4. Re-export from store/index.ts ────────────────────────────────────────

    Add: `export { useMoodEntryStore } from './moodEntryStore';`
