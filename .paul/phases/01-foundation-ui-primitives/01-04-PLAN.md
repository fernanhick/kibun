## Goal
Build the MoodBubble shared component — a tappable, color-coded circle that renders any of the 14 moods from Plan 01-03's palette. Implement three size variants, a spring-animated selection state, pressed feedback, disabled state, and full accessibility. Also resolve the tab layout color debt documented in Plan 01-03.

## Purpose
MoodBubble is the core interactive UI element of the entire app. It appears in:
- Phase 2 / Plan 02-01: `FirstMoodScreen` — hero-size single bubble prompt
- Phase 4 / Plan 04-01: `MoodSelectionScreen` — full 14-bubble grid
- Phase 5 / Plan 05-02: `HistoryScreen` — calendar day color indicators (via `tintColor`)

Building it once here as a dumb, reusable primitive (no data fetching, no state beyond selection) means all three phases just import and wire props.

<task type="auto">
  <name>Task 1: Create MoodBubble component</name>
  <files>
    src/components/MoodBubble.tsx
  </files>
  <action>
    Create `src/components/MoodBubble.tsx`:

    **Props interface:**
    ```typescript
    interface MoodBubbleProps {
      mood: MoodDefinition;
      size?: 'sm' | 'md' | 'lg';
      selected?: boolean;
      onPress?: (mood: MoodDefinition) => void;
      disabled?: boolean;
    }
    ```

    **Size map** (diameter in pt):
    ```typescript
    const SIZES = { sm: 48, md: 72, lg: 96 } as const;
    ```

    **Font size per size:**
    ```
    sm → typography.sizes.xs  (11pt)
    md → typography.sizes.sm  (13pt)
    lg → typography.sizes.md  (15pt)
    ```

    **Animation:**
    - Use `Animated.Value` from `react-native` (NOT react-native-reanimated — not yet installed)
    - Initialize: `const scaleAnim = useRef(new Animated.Value(1)).current`
    - On `selected` change via `useEffect` with dependency array `[selected]`:
      ```typescript
      useEffect(() => {
        const animation = Animated.spring(scaleAnim, {
          toValue: selected ? 1.08 : 1,
          useNativeDriver: true,
          tension: 300,
          friction: 10,
        });
        animation.start();
        return () => animation.stop(); // Stop on unmount or before next effect run
      }, [selected]);
      ```
    - The cleanup `() => animation.stop()` is required: if the component unmounts or selected
      changes while the spring is mid-flight, the animation fires against an unmounted node.

    **Structure:**
    ```tsx
    <Animated.View style={[styles.wrapper, { transform: [{ scale: scaleAnim }] }]}>
      <Pressable
        onPress={disabled ? undefined : () => onPress?.(mood)}
        hitSlop={{ top: 8, bottom: 8, left: 8, right: 8 }}
        style={({ pressed }) => [
          styles.bubble,
          sizeStyle,
          { backgroundColor: mood.bubbleColor },
          selected && styles.selected,
          disabled && styles.disabled,
          pressed && !disabled && styles.pressed,
        ]}
        accessibilityRole={onPress ? 'button' : 'text'}
        accessibilityLabel={mood.label}
        accessibilityState={onPress ? { selected, disabled } : undefined}
      >
        <Text style={[styles.label, fontSizeStyle, { color: mood.textColor }]}>
          {mood.label}
        </Text>
      </Pressable>
    </Animated.View>
    ```
    Note: `hitSlop` extends the touchable area 8pt beyond the visual circle on all sides without
    affecting layout. `accessibilityRole="text"` (not "button") when `onPress` is undefined — a
    non-interactive element with role="button" confuses VoiceOver/TalkBack users.

    **StyleSheet definitions:**
    - `bubble`: `borderRadius: radius.bubble`, `alignItems: 'center'`, `justifyContent: 'center'`, `...shadows.sm`
    - `selected`: `...shadows.md` (elevated shadow replaces sm — do not stack)
    - `pressed`: `opacity: 0.82`
    - `disabled`: `opacity: 0.38`
    - `label`: `fontWeight: typography.weights.semibold`, `textAlign: 'center'`
    - `wrapper`: `alignSelf: 'flex-start'` — prevents Animated.View from stretching in flex parent

    **Imports required:**
    - `Animated`, `Pressable`, `Text`, `StyleSheet` from `react-native`
    - `useRef`, `useEffect` from `react`
    - `MoodDefinition` from `@constants/moods`
    - `colors`, `typography`, `radius`, `shadows` from `@constants/theme`

    Avoid: Importing from `react-native-reanimated` — not yet installed; Reanimated arrives in Plan 01-05
    Avoid: Storing selection state inside MoodBubble — `selected` is controlled externally by the screen
    Avoid: `useNativeDriver: false` — always true for transform animations to keep off JS thread
    Avoid: Hardcoded pixel values — use SIZES map and typography tokens
  </action>
