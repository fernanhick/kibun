<objective>
## Goal
Add on-device pattern detection that surfaces simple, actionable observations from mood data — day-of-week patterns, time-of-day patterns, and mood trend direction — displayed as text flags on the InsightsScreen.

## Purpose
Pattern flags deliver the core value proposition: "insights that reveal patterns they wouldn't notice themselves." Charts show data; patterns interpret it. A user who sees "You often feel Tired on Mondays" gets an immediate aha-moment that drives continued engagement. This is the differentiator between a mood logger and a mood insights app.

## Output
- `src/lib/patterns.ts` — pure pattern detection functions (no ML, no external deps)
- `app/(tabs)/insights.tsx` — pattern flags section added below charts
</objective>

<task type="auto">
  <name>Task 1: Create pattern detection utilities</name>
  <files>src/lib/patterns.ts</files>
  <action>
    Create src/lib/patterns.ts with pure pattern detection functions:

    1. PatternFlag interface:
       ```typescript
       export interface PatternFlag {
         id: string;        // unique key for React list rendering
         icon: string;      // emoji for visual distinction (e.g., "📅", "🕐", "📈")
         text: string;      // human-readable pattern description
         type: 'day-of-week' | 'time-of-day' | 'trend';
       }
       ```

    2. SLOT_LABELS constant (typed as Record<MoodSlot, string> for exhaustiveness checking): <!-- audit-added: compile-time error if MoodSlot grows -->
       Map MoodSlot to human-readable strings for pattern text:
       - 'morning' → 'the morning'
       - 'afternoon' → 'the afternoon'
       - 'night' → 'the evening'
       - 'pre_sleep' → 'before sleep'

    3. DAY_LABELS constant:
       Array of weekday names: ['Sunday', 'Monday', 'Tuesday', 'Wednesday', 'Thursday', 'Friday', 'Saturday']
       (matches JS Date.getDay() index)

    4. detectDayOfWeekPatterns(entries: MoodEntry[]): PatternFlag[]
       - For each moodId in entries, compute:
         a. Overall frequency: count(moodId) / total entries
         b. Per-weekday frequency: count(moodId on day) / count(all entries on day)
         c. Guard: skip any day where count(all entries on day) === 0 to prevent division by zero <!-- audit-added: 0 entries on a day produces Infinity ratio, falsely triggering pattern -->
       - If a mood's weekday frequency exceeds 1.5x its overall frequency AND the mood appears at least 3 times on that day:
         Create flag: "You often feel {label} on {day}s"
       - id: `dow-{moodId}-{dayIndex}`
       - icon: "📅"
       - Limit to top 2 most significant patterns (highest ratio)
       - Sort by ratio descending before limiting

    5. detectTimeOfDayPatterns(entries: MoodEntry[]): PatternFlag[]
       - For each moodId in entries, compute:
         a. Overall frequency: count(moodId) / total entries
         b. Per-slot frequency: count(moodId in slot) / count(all entries in slot)
         c. Guard: skip any slot where count(all entries in slot) === 0 to prevent division by zero <!-- audit-added: same 0-denominator risk as day-of-week -->
       - If a mood's slot frequency exceeds 1.5x its overall frequency AND the mood appears at least 3 times in that slot:
         Create flag: "You tend to feel {label} in {slotLabel}"
       - id: `tod-{moodId}-{slot}`
       - icon: "🕐"
       - Limit to top 2
       - Sort by ratio descending before limiting

    6. detectTrendPattern(entries: MoodEntry[]): PatternFlag | null
       - MUST sort entries by loggedAt ascending before splitting — store prepends new entries (reverse-chronological), so unsorted split produces wrong halves <!-- audit-added: moodEntryStore.addEntry uses [entry, ...state.entries] -->
       - Split sorted entries into first half and second half
       - Compute average GROUP_SCORE for each half (using MOOD_MAP group lookup + GROUP_SCORES)
       - If second half average - first half average >= 0.5: "Your mood has been improving lately 🌱"
       - If first half average - second half average >= 0.5: "Your mood has been dipping recently"
       - Otherwise: null (no significant trend)
       - id: 'trend'
       - icon: "📈" for improving, "📉" for declining
       - type: 'trend'

    7. detectPatterns(entries: MoodEntry[]): PatternFlag[]
       - Main entry point that combines all detectors
       - If entries.length < 7: return [] (minimum threshold)
       - Call all three detectors, concatenate results
       - Return combined array (max ~5 flags total)

    Imports: MoodEntry, MoodSlot from @models/index; MOOD_MAP from @constants/moods; GROUP_SCORES from @lib/insights

    Avoid: Do NOT import stores. Do NOT use any ML/TF libraries. Do NOT import charting library.
    Avoid: Do NOT use complex statistical methods — simple ratio comparison is sufficient for MVP.
  </action>
  <verify>npx tsc --noEmit exits 0; patterns.ts exports PatternFlag, detectPatterns, and the three individual detectors</verify>
  <done>Pattern detection utilities ready. AC-6 partial: zero TS errors.</done>
