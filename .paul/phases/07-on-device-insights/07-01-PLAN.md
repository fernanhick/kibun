  <name>Task 1: Install charting library and create insights data utilities</name>
  <files>src/lib/insights.ts</files>
  <action>
    1. Install dependencies:
       ```
       npx expo install react-native-svg
       npm install react-native-gifted-charts
       ```
       react-native-svg must be installed via `npx expo install` for SDK version compatibility.
       react-native-gifted-charts uses react-native-svg for rendering — no Skia dependency.

    2. Create src/lib/insights.ts with these pure utility functions:

       a. GROUP_SCORES constant:
          Map MoodGroup to numeric score for trend calculations:
          - 'green' → 4 (positive)
          - 'neutral' → 3
          - 'blue' → 2 (introspective)
          - 'red-orange' → 1 (negative)
          Export as `const GROUP_SCORES: Record<MoodGroup, number>`

       b. filterEntriesByDays(entries: MoodEntry[], days: number): MoodEntry[]
          - Compute cutoff date string using UTC-consistent approach: <!-- audit-added: local midnight via setHours(0,0,0,0) would create timezone-dependent boundary -->
            ```
            const cutoff = new Date();
            cutoff.setDate(cutoff.getDate() - days);
            const cutoffStr = cutoff.toISOString();
            ```
            Do NOT use setHours(0,0,0,0) — that sets local midnight, but loggedAt is stored via toISOString() (UTC). Converting local midnight to ISO shifts the boundary by timezone offset, silently including/excluding entries near day boundaries.
          - Return entries where loggedAt >= cutoffStr (lexicographic ISO comparison is correct for UTC ordering)
          - Pure function, no side effects

       c. getMoodFrequency(entries: MoodEntry[]): { moodId: string; label: string; color: string; count: number }[]
          - Count occurrences of each moodId across entries
          - For each moodId, look up MOOD_MAP for label and bubbleColor
          - Sort by count descending
          - Return top results (no limit — screen can slice)
          - If moodId not in MOOD_MAP (defensive), use label=moodId, color='#BDBDBD'

       d. getDailyMoodScores(entries: MoodEntry[]): { date: string; score: number; label: string }[]
          - Group entries by date (loggedAt.split('T')[0])
          - For each date, compute average score:
            - For each entry, get mood group via MOOD_MAP[entry.moodId]?.group
            - Map group to score via GROUP_SCORES
            - Average all scores for that date
          - label: format date as short string (e.g. "Apr 3" or just the day number)
          - Sort by date ascending (chronological for line chart)
          - If a mood is not in MOOD_MAP, default to score 3 (neutral)

    Imports: MoodEntry from @models/index, MOOD_MAP and MoodGroup from @constants/moods
    
    3. After installing react-native-gifted-charts, verify its TypeScript types resolve: <!-- audit-added: library may ship incomplete types -->
       - Run `npx tsc --noEmit` after install
       - If BarChart/LineChart imports produce "Could not find declaration file" errors, create `src/types/react-native-gifted-charts.d.ts` with `declare module 'react-native-gifted-charts';`
       - This is a fallback only — check installed version first, types may resolve cleanly

    Imports: MoodEntry from @models/index, MOOD_MAP and MoodGroup from @constants/moods
    
    Avoid: Do NOT import any store — these utilities are pure functions that accept entries as a parameter (same pattern as notifications.ts).
    Avoid: Do NOT import the charting library here — data utilities are library-agnostic.
  </action>
