<objective>
## Goal
Build the first three profile collection screens (Personal, Work, Physical) with a reusable OptionPicker component and a Zustand onboardingStore that accumulates profile data across screens without resetting.

## Purpose
The AI insight engine (Phase 8) uses lifestyle context — age, work style, sleep, exercise — to detect patterns that mood alone can't explain ("You feel Tired most often after <6h sleep"). These screens collect that context during onboarding, before the paywall gate. Data is held in onboardingStore and written to Supabase in Plan 02-03 when the full profile is complete.

## Output
- `src/types/index.ts` — OnboardingProfile type added
- `src/store/onboardingStore.ts` — Zustand store accumulating profile across screens
- `src/components/OptionPicker.tsx` — reusable chip-style single-select component
- `src/components/index.ts` — OptionPicker exported
- `app/(onboarding)/profile-personal.tsx` — name (TextInput) + age range + gender
- `app/(onboarding)/profile-work.tsx` — employment status + conditional work setting + hours
- `app/(onboarding)/profile-physical.tsx` — sleep hours + exercise frequency
- `app/(onboarding)/profile-social.tsx` — stub (real screen in 02-03)
</objective>

    **src/store/onboardingStore.ts — new file**
    Create a Zustand store (no persistence — onboarding data is transient until written to Supabase in 02-03):
    ```ts
    import { create } from 'zustand';
    import { OnboardingProfile } from '@models/index';

    const INITIAL_PROFILE: OnboardingProfile = {
      name: '',
      ageRange: null,
      gender: null,
      employment: null,
      workSetting: null,
      workHours: null,
      sleepHours: null,
      exercise: null,
      socialFrequency: null,
      stressLevel: null,
      goals: [],
    };

    interface OnboardingState {
      profile: OnboardingProfile;
      updateProfile: (patch: Partial<OnboardingProfile>) => void;
      resetProfile: () => void;
    }

    export const useOnboardingStore = create<OnboardingState>()((set) => ({
      profile: INITIAL_PROFILE,
      updateProfile: (patch) =>
        set((state) => ({ profile: { ...state.profile, ...patch } })),
      resetProfile: () => set({ profile: INITIAL_PROFILE }),
    }));
    ```
    Note: No `persist` middleware — profile is written to Supabase at onboarding completion (02-03). In-memory only avoids stale data on reinstall.

    **src/components/OptionPicker.tsx — new file**
    Create a chip-style single-select component:
    - Props:
      ```ts
      interface OptionPickerProps {
        label: string;
        options: PickerOption[];
        selected: string | null;
        onSelect: (value: string) => void;
        accessibilityLabel?: string;
      }
      ```
    - Imports: View, Text, Pressable, StyleSheet from react-native; PickerOption from @models/index; colors, typography, spacing, radius from @constants/theme
    - Layout: label Text above, then a View with flexDirection='row', flexWrap='wrap', gap=spacing.sm
    - Each option: Pressable with hitSlop={{ top: 6, bottom: 6, left: 6, right: 6 }}
      - Selected: backgroundColor=colors.primary, border transparent
      - Unselected: backgroundColor=colors.surface, borderWidth=1.5, borderColor=colors.border
      - Text: selected → colors.textInverse; unselected → colors.text
      - fontSize: typography.sizes.sm, fontWeight: typography.weights.medium
      - paddingVertical: spacing.sm, paddingHorizontal: spacing.md, borderRadius: radius.full
    - Accessibility: each Pressable gets accessibilityRole='radio', accessibilityState={{ checked: isSelected }}, accessibilityLabel=option.label
      // AUDIT M1: 'radio' role requires accessibilityState.checked (not .selected) — checked is the
      // semantically correct companion for radio per WCAG, React Native docs, and iOS/Android a11y
    - The wrapper View gets accessibilityRole='radiogroup' and accessibilityLabel (from prop or label)
    - Do NOT use FlatList — use options.map() (static lists, no virtualization needed)
