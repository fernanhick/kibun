<objective>
## Goal
Update paywall navigation to route successful purchasers to a RegistrationScreen, build the full registration UI shell (Apple / Google / email form / skip), and add an anonymous data-loss warning banner to the HomeScreen.

## Purpose
Plan 03-01 built the paywall but sent all users — including new subscribers — directly to tabs. Subscribers need an immediate prompt to create an account so their data isn't device-bound. This plan wires that path and surfaces the anonymous data-loss risk for users who skip. Actual Supabase OAuth calls are stubbed here and implemented in Plan 03-03 (requires external provider setup and native module rebuild).

## Output
- `app/paywall.tsx` — handlePurchase now routes successful purchase to `/register`; dev/error path unchanged → `/(tabs)`
- `app/_layout.tsx` — `register` Stack.Screen registered

- `app/(tabs)/index.tsx` — anonymous data-loss banner visible when `authStatus === 'anonymous'`
</objective>

    ── 1. app/paywall.tsx ────────────────────────────────────────────────────────

    Add a `purchased` flag to distinguish the success path from dev fallback / errors.
    The current code uses a single fall-through for all non-cancel paths. Replace with:

    ```ts
    const handlePurchase = async () => {
      if (purchasing) return;
      setPurchasing(true);
      let purchased = false;
      try {
        const offerings = await Purchases.getOfferings();
        const pkg = offerings.current?.availablePackages[0];
        if (!pkg) {
          if (__DEV__) {
            console.warn('[kibun:rc] No offerings found. Configure products in RevenueCat dashboard.');
          }
          // No products — fall through to tabs (dev fallback, not a real purchase)
        } else {
          const { customerInfo } = await Purchases.purchasePackage(pkg);
          const active = customerInfo.entitlements.active['premium'];
          if (active) {
            setSubscriptionStatus(active.periodType === 'TRIAL' ? 'trial' : 'active');
            purchased = true; // Only set on confirmed entitlement
          }
        }
      } catch (error: unknown) {
        const rcError = error as { code?: string };
        if (rcError?.code === PURCHASES_ERROR_CODE.PURCHASE_CANCELLED_ERROR) {
          setPurchasing(false);
          return;
        }
        if (__DEV__) {
          console.warn('[kibun:rc] Purchase failed (non-cancel):', error);
        }
      }
      setPaywallSeen();
      // Successful purchase → registration; dev/error fallback → tabs directly
      router.replace(purchased ? '/register' : '/(tabs)');
    };
    ```

    Avoid: routing to /register on the dev fallback path — that would show registration
    to users who haven't actually purchased, breaking the intended flow.
