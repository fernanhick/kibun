<objective>
## Goal
Create the Supabase Edge Function that generates AI mood reports via OpenAI, the database migration for report storage, and a client-side service for requesting and fetching reports.

## Purpose
Cloud AI reports are the premium feature that justifies the subscription. While on-device pattern detection (Phase 7) provides basic flags, the OpenAI-powered reports deliver narrative, personalized insights — "Your mood drops on Monday afternoons, likely correlated with your work schedule. Consider..." — that users can't get elsewhere. This is the revenue-generating differentiator.

## Output
- `supabase/migrations/20260405_create_ai_reports.sql` — report storage table
- `supabase/functions/generate-report/index.ts` — Deno Edge Function (OpenAI integration)
- `src/lib/aiReports.ts` — client-side service for requesting/fetching reports
- `src/types/index.ts` — AIReport type added
</objective>

## AC-1: Edge Function generates report via OpenAI
```gherkin
Given a registered user with mood entries in the database
When the generate-report Edge Function is invoked with a valid JWT, report_type, and optional profile context
Then it authenticates the user from the JWT, queries mood_entries for the period, calls OpenAI API, and returns a structured report
And returns 401 if the JWT is missing or invalid
```

## AC-2: AI reports table stores generated reports
```gherkin
Given the Edge Function generates a report
When the report is stored
Then the ai_reports table contains: user_id, report_type (weekly/monthly), period_start, period_end, content (narrative text), mood_summary (JSON), created_at
```

## AC-3: Client service can request reports
```gherkin
Given a registered user in the app
When the client calls requestReport() with reportType and optional profile
Then it invokes the Edge Function via supabase.functions.invoke() (which auto-sends the user's JWT)
And the Edge Function authenticates the user from the JWT, not from a client-supplied user_id
```

## AC-4: Client service can fetch existing reports
```gherkin
Given reports exist for a user
When the client calls getReports()
Then it returns reports from the ai_reports table ordered by created_at descending
```

  <name>Task 1: Create Supabase migration and Edge Function</name>
  <files>supabase/migrations/20260405_create_ai_reports.sql, supabase/functions/generate-report/index.ts</files>
  <action>
    1. Create supabase/ directory structure:
       - supabase/migrations/ (for SQL migrations)
       - supabase/functions/generate-report/ (for Edge Function)

    2. Create migration file `supabase/migrations/20260405_create_ai_reports.sql`:
       ```sql
       create table if not exists public.ai_reports (
         id uuid primary key default gen_random_uuid(),
         user_id uuid references auth.users(id) on delete cascade not null,
         report_type text not null check (report_type in ('weekly', 'monthly')),
         period_start date not null,
         period_end date not null,
         content text not null,
         mood_summary jsonb default '{}',
         created_at timestamptz default now() not null
       );

       -- Index for fetching user reports efficiently
       create index if not exists idx_ai_reports_user_id on public.ai_reports(user_id, created_at desc);

       -- RLS: users can only read their own reports
       alter table public.ai_reports enable row level security;

       create policy "Users can read own reports"
         on public.ai_reports for select
         using (auth.uid() = user_id);

       -- Edge Function inserts via service_role key (bypasses RLS)
       -- No insert policy needed for regular users
       ```

    3. Create Edge Function `supabase/functions/generate-report/index.ts` (Deno runtime):

       The function should:
       a. Use Deno.serve() natively (do NOT import serve from deno.land/std — deprecated for Supabase Edge Functions) <!-- audit-added: MH-2 Supabase Edge Functions use Deno.serve() since Deno 1.35+ -->
       b. Import: createClient from "https://esm.sh/@supabase/supabase-js@2"

       c. Handle POST request with JSON body:
          - report_type: 'weekly' | 'monthly' (required)
          - profile: { name, ageRange, employment, workSetting, sleepHours, exercise, socialFrequency, stressLevel, goals } (optional — sent from client since profile isn't persisted server-side)
          - NOTE: user_id is NOT accepted from the body — see step (d2) <!-- audit-added: MH-1 -->

       d. Create Supabase admin client using Deno.env.get('SUPABASE_URL') and Deno.env.get('SUPABASE_SERVICE_ROLE_KEY')

       d2. MUST extract user identity from Authorization header JWT — do NOT trust client-supplied user_id in the request body. Use the admin client to verify: `const { data: { user }, error } = await adminClient.auth.getUser(jwt)` where jwt is extracted from `req.headers.get('Authorization')?.replace('Bearer ', '')`. Return 401 if no valid user. This prevents any authenticated user from generating reports for another user. <!-- audit-added: MH-1 critical auth validation -->

       e. Query mood_entries for the user:
          - weekly: last 7 days
          - monthly: last 30 days
          - Select: mood, note, check_in_slot, logged_at
          - Order by logged_at ascending

       f. If no entries found: return 200 with { report: null, reason: 'no_entries' }

       g. Build OpenAI prompt:
          - System message: "You are a warm, insightful mood analyst for the kibun app. Generate a personalized mood report. Be supportive, specific, and actionable. Use the user's name if provided. Keep the report concise (200-300 words). Structure as: 1) Summary of mood patterns, 2) Notable observations, 3) One gentle, actionable suggestion."
          - User message: Include:
            - Report type and period
            - Mood data as a compact list: "[date] [slot]: [mood] (note: [note])" for each entry
            - Profile context if provided (age range, employment, sleep habits, etc.)
            - "Generate a {report_type} mood report for {name or 'this user'}."

       h. Call OpenAI API:
          - URL: https://api.openai.com/v1/chat/completions
          - Model: gpt-4o-mini
          - API key from Deno.env.get('OPENAI_API_KEY')
          - Temperature: 0.7
          - max_tokens: 500

       i. Parse OpenAI response, extract content from choices[0].message.content

       j. Build mood_summary JSON from the entries:
          - Total entries in period
          - Top 3 moods by frequency
          - Average entries per day

       k. Insert into ai_reports table via admin client:
          - user_id, report_type, period_start, period_end, content, mood_summary

       l. Return 200 with the full report object

       m. Error handling:
          - Missing required fields: return 400 with { error: 'missing_fields' }
          - Invalid/missing JWT: return 401 with { error: 'unauthorized' } <!-- audit-added: MH-1 -->
          - OpenAI API failure: return 502 with { error: 'ai_unavailable' } — do NOT forward raw OpenAI error details to client (may contain API key fragments or internal info) <!-- audit-added: MH-3 sanitize errors -->
          - Database error: return 500 with { error: 'storage_failed' }
          - Wrap in try/catch at top level — catch-all returns 500 with generic { error: 'internal_error' }, logs details server-side only

       m2. Duplicate report prevention: Before calling OpenAI, check if an ai_reports row already exists for this user_id + report_type with created_at within the current period (last 7 days for weekly, last 30 days for monthly). If a recent report exists, return it directly instead of generating a new one. This prevents accidental duplicate OpenAI API calls (each costs money). <!-- audit-added: SR-1 rate limiting -->

       n. CORS headers: Include Access-Control-Allow-Origin, allow OPTIONS preflight

    Avoid: Do NOT use npm imports in Edge Function — Deno uses URL imports (esm.sh, deno.land)
    Avoid: Do NOT use supabase-js v1 patterns — use v2 createClient
    Avoid: Do NOT hardcode API keys — use Deno.env.get() for all secrets
  </action>

    1. Add AIReport interface to src/types/index.ts (after MoodEntry):
       ```typescript
       export interface AIReport {
         id: string;
         userId: string;
         reportType: 'weekly' | 'monthly';
         periodStart: string;  // ISO date
         periodEnd: string;    // ISO date
         content: string;      // narrative text from OpenAI
         moodSummary: {
           totalEntries: number;
           topMoods: { moodId: string; count: number }[];
           avgEntriesPerDay: number;
         } | null;
         createdAt: string;    // ISO timestamp
       }
       ```

    2. Create src/lib/aiReports.ts with pure service functions:

       a. Import: supabase from @lib/supabase
       b. Import: AIReport from @models/index
       c. Import: OnboardingProfile from @models/index

       d. Helper: mapRow(row: any): AIReport
          - Maps snake_case DB columns to camelCase AIReport fields:
            id, user_id→userId, report_type→reportType, period_start→periodStart,
            period_end→periodEnd, content, mood_summary→moodSummary, created_at→createdAt

       e. requestReport(params: { reportType: 'weekly' | 'monthly'; profile?: Partial<OnboardingProfile> }): Promise<AIReport | null> <!-- audit-modified: removed userId param — Edge Function extracts user from JWT, supabase client auto-sends Authorization header -->
          - Calls supabase.functions.invoke('generate-report', { body: { report_type: params.reportType, profile: params.profile } })
          - supabase.functions.invoke() returns { data, error } where data is the parsed JSON response body <!-- audit-added: SR-2 clarify invoke response -->
          - If error: console.error in __DEV__, return null
          - If data is null or data.report is null (no entries): return null
          - Map data (the report object from Edge Function response) to AIReport and return

       f. getReports(userId: string): Promise<AIReport[]>
          - Queries supabase.from('ai_reports').select('*').eq('user_id', userId).order('created_at', { ascending: false }).limit(10)
          - If error: console.error in __DEV__, return []
          - Map rows via mapRow, return array

       g. getLatestReport(userId: string, reportType: 'weekly' | 'monthly'): Promise<AIReport | null>
          - Queries supabase.from('ai_reports').select('*').eq('user_id', userId).eq('report_type', reportType).order('created_at', { ascending: false }).limit(1).maybeSingle()
          - If error or no data: return null
          - Map and return

    Imports: supabase from @lib/supabase; AIReport, OnboardingProfile from @models/index

    Avoid: Do NOT import stores — service functions accept parameters
    Avoid: Do NOT add loading/error state management — that's for the screen (08-02)
    Avoid: Do NOT add caching — simple fetch-on-demand for MVP
