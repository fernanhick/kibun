<objective>
## Goal
Replace the placeholder theme.ts with a full design token system, define the complete 14-mood color palette with WCAG 2.1 AA compliance, and create three shared component primitives — Button, Card, Screen — that all subsequent phases use directly.

## Purpose
Plans 01-04 (MoodBubble) and 01-05 (Lottie/SplashScreen), and all of Phase 2 (Onboarding), import from this token system and these components. Building them once here prevents each screen from inventing its own spacing, colors, or button styles. The mood color palette in particular is a shared contract: Plan 01-04 uses it for bubbles, Phase 5 uses it for calendar day tints, Phase 7 uses it for trend charts.

## Output
- `src/constants/theme.ts` — full token system: colors, typography, spacing, radius, shadows
- `src/constants/moods.ts` — 14 mood definitions each with bubbleColor, textColor, tintColor (WCAG AA verified)
- `src/components/Button.tsx` — primary / secondary / ghost variants, accessible
- `src/components/Card.tsx` — container with standard padding and surface styling
- `src/components/Screen.tsx` — SafeAreaView + optional scroll, standard padding
- `src/components/index.ts` — re-export barrel
</objective>

    1. Create `src/components/Button.tsx`:
       - Props interface:
         ```typescript
         interface ButtonProps {
           label: string;
           onPress: () => void;
           variant?: 'primary' | 'secondary' | 'ghost';
           size?: 'md' | 'sm';
           disabled?: boolean;
           loading?: boolean;
           fullWidth?: boolean;
           accessibilityHint?: string;
         }
         ```
       - Use `Pressable` (not TouchableOpacity) — Pressable supports the pressed state style callback and is the React Native recommended pattern
       - Variant styling:
         - `primary`: background `colors.primary`, text `colors.textInverse`, radius `radius.button`
         - `secondary`: background `colors.primaryLight`, text `colors.primary`, radius `radius.button`
         - `ghost`: background transparent, text `colors.primary`, border 1.5px `colors.primary`
       - Size styling:
         - `md`: paddingVertical 14, paddingHorizontal 24, fontSize `typography.sizes.md`, fontWeight `typography.weights.semibold`
         - `sm`: paddingVertical 10, paddingHorizontal 16, fontSize `typography.sizes.sm`, fontWeight `typography.weights.semibold`
       - Disabled state: opacity 0.4, `accessibilityState={{ disabled: disabled || loading }}`
       - Pressed state: opacity 0.85 (via Pressable style callback) — suppress when loading or disabled
       - Loading state: when `loading={true}`, render `<ActivityIndicator size="small" color={textColor} />` in place of the label text; block `onPress` (treat as disabled)
       - Accessibility: `accessibilityRole="button"`, `accessibilityLabel={label}`, `accessibilityHint={accessibilityHint}` (passthrough — context-specific hints are supplied by the screen, not the component)

    2. Create `src/components/Card.tsx`:
       - Props interface:
         ```typescript
         interface CardProps {
           children: React.ReactNode;
           style?: StyleProp<ViewStyle>;
           padding?: keyof typeof spacing;
         }
         ```
       - Default: background `colors.surfaceElevated`, radius `radius.card`, padding `spacing.md`, shadow `shadows.sm`
       - `style` prop is spread last to allow overrides

    3. Create `src/components/Screen.tsx`:
       - Props interface:
         ```typescript
         interface ScreenProps {
           children: React.ReactNode;
           scrollable?: boolean;
           style?: StyleProp<ViewStyle>;
           contentContainerStyle?: StyleProp<ViewStyle>;
         }
         ```
       - Import `SafeAreaView` from `react-native-safe-area-context` — NOT from `react-native`.
         The project wraps the app in `SafeAreaProvider` (from `react-native-safe-area-context`) in `app/_layout.tsx`.
         `SafeAreaView` from `react-native` does not consume this context and will produce wrong insets on notched iOS devices without crashing (silent failure).
       - Outer: `SafeAreaView` (from `react-native-safe-area-context`) with `flex: 1` and `backgroundColor: colors.background`
       - If `scrollable={true}`: wrap children in `ScrollView` with `showsVerticalScrollIndicator={false}`
       - Default horizontal padding: `spacing.screenPadding` (20)
       - `contentContainerStyle` forwarded to ScrollView when scrollable

    4. Create `src/components/index.ts`:
       ```typescript
       export { Button } from './Button';
       export { Card } from './Card';
       export { Screen } from './Screen';
       ```
       Named exports only — no default exports.

    Avoid: Barrel-importing from @components (import directly from @components/Button, @components/Card, @components/Screen where tree-shaking matters)
    Avoid: Nesting SafeAreaView inside SafeAreaView — Screen handles the safe area; screens that use Screen must not add their own SafeAreaView
    Avoid: Platform-specific shadow hacks — use the shadow tokens from theme.ts which work cross-platform
