<task type="auto">
  <name>Task 2: moodResponses constants + MoodResponseScreen</name>
  <files>src/constants/moodResponses.ts, app/(onboarding)/mood-response.tsx</files>
  <action>
    **src/constants/moodResponses.ts**
    Create a record mapping each MoodId to a short, warm, non-clinical phrase (1–2 sentences max).
    Export with compile-time exhaustiveness check: <!-- audit-added: strongly-recommended #2 — satisfies catches missing keys at compile time -->
    `export const MOOD_RESPONSES = { ... } satisfies Record<MoodId, string>`
    Do NOT use a plain type annotation (`const x: Record<MoodId, string>`) — `satisfies` catches missing MoodId keys at compile time while preserving the literal type for type inference. A plain annotation would allow a missing key to silently produce `undefined` at runtime.

    Write one phrase per mood. Tone: warm, curious, never judgmental. Examples:
    - happy: "That's wonderful! Let's hold onto that feeling."
    - excited: "Big energy! Something good is happening."
    - grateful: "Gratitude is powerful — you're already practicing it."
    - calm: "Peaceful moments are worth noticing."
    - meh: "Not every day needs to sparkle. You showed up."
    - tired: "Rest is productive. Your body is telling you something."
    - bored: "Boredom is creativity waiting for a spark."
    - confused: "Uncertainty means you're thinking — that's a good sign."
    - sad: "It's okay to feel this way. You're not alone."
    - anxious: "Take a breath. You're here, and that's enough."
    - frustrated: "Friction means you care. That matters."
    - angry: "Strong feelings deserve space. Let's track them."
    - melancholy: "A quiet ache — sometimes hard to name, always valid."
    - lonely: "Connection takes courage. Noticing the feeling is the first step."

    These are first-pass copy — content is easy to iterate, structure is what matters.

    **app/(onboarding)/mood-response.tsx**
    Build MoodResponseScreen:
    - Imports: Screen, Button (from @components), Shiba, ShibaVariant (from @components/Shiba), MoodBubble (from @components), MOOD_MAP, MoodId, MoodGroup (from @constants/moods), MOOD_RESPONSES (from @constants/moodResponses) <!-- audit-added: must-have #2 — MoodGroup required for shibaVariant type annotation -->
    - Import: useLocalSearchParams, useRouter from expo-router

    Read route param: `const { moodId } = useLocalSearchParams<{ moodId: MoodId }>()`
    Look up mood: `const mood = MOOD_MAP[moodId]`

    Define shibaVariant as a MODULE-LEVEL pure function (outside the component): <!-- audit-added: strongly-recommended #3 — impure closure avoided -->
    ```
    function shibaVariant(group: MoodGroup, id: MoodId): ShibaVariant {
      if (group === 'green') return id === 'excited' ? 'excited' : 'happy';
      if (group === 'neutral') return 'neutral';
      return 'sad'; // red-orange + blue
    }
    ```
    Inside the component: `const variant = mood ? shibaVariant(mood.group, mood.id) : 'neutral';`

    Layout (Screen, flex column, centered content):
    1. Shiba — variant, size=140, loop=false, autoPlay=true (plays once on mount, signals reaction)
    2. MoodBubble — mood={mood}, size='lg', disabled={true} — centered, marginTop spacing.md
    3. Phrase Text — MOOD_RESPONSES[moodId] — typography.sizes.md, color colors.textSecondary, textAlign center, paddingHorizontal spacing.lg, marginTop spacing.md
    4. "Continue" Button — onPress: `router.push('/(onboarding)/profile-personal')` — marginTop spacing.xl, marginHorizontal spacing.md

    Avoid: crashing if moodId is undefined — add guard: `if (!mood) return null`
    Avoid: passing MoodDefinition objects as params — only the string moodId crosses the route boundary.
  </action>
