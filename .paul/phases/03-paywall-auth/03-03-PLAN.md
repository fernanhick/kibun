<objective>
## Goal
Replace all TODO stubs in RegistrationScreen with real Supabase auth calls: email/password signup, Google OAuth, and Apple Sign In. Anonymous users are converted to registered accounts where possible (email and Google preserve the anonymous userId; Apple creates a new registered session). Error states and cancellation are handled gracefully.

## Purpose
Plans 03-01 and 03-02 built the paywall + RegistrationScreen UI shell but left all three auth handlers as stubs. This plan wires the real auth calls, completing Phase 3. After this plan, a new subscriber who reaches RegistrationScreen can create a real account, their session.authStatus becomes 'registered', and the anonymous data-loss banner disappears.

- `app.config.ts` — expo-apple-authentication config plugin added
</objective>

    ── handleEmail ───────────────────────────────────────────────────────────────

    Converts the anonymous session to a registered email/password account.
    `supabase.auth.updateUser` preserves the anonymous userId — same user, now registered.

    ```ts
    const handleEmail = async () => {
      if (!email.trim() || !password.trim() || submitting) return;
      setSubmitting(true);
      setError(null);
      const { error: authError } = await supabase.auth.updateUser({
        email: email.trim(),
        password: password.trim(),
      });
      if (authError) {
        setError(authError.message);
        setSubmitting(false);
        return;
      }
      // updateUser dispatches a confirmation email.
      // If email confirmation is disabled in Supabase dashboard: session upgrades immediately.
      // If enabled: onAuthStateChange fires when the user clicks the confirmation link.
      // Navigate to tabs in both cases — the banner disappears once the session upgrades.
      router.replace('/(tabs)');
    };
    ```

    ── handleGoogle ──────────────────────────────────────────────────────────────

    `linkIdentity` links Google to the existing anonymous user, preserving userId.
    `skipBrowserRedirect: true` gives us the URL to open manually via WebBrowser.

    ```ts
    const handleGoogle = async () => {
      setError(null);
      const redirectTo = makeRedirectUri({ scheme: 'kibun', path: 'auth/callback' });
      const { data, error: authError } = await supabase.auth.linkIdentity({
        provider: 'google',
        options: { redirectTo, skipBrowserRedirect: true },
      });
      if (authError || !data?.url) { // audit-added M1: data?.url — data can be null
        setError(authError?.message ?? 'Google sign in unavailable');
        return;
      }
      try { // audit-added S2: WebBrowser can throw on native module failure
        const result = await WebBrowser.openAuthSessionAsync(data.url, redirectTo);
        if (result.type !== 'success') return; // Cancelled — no error shown, stays on screen
        const { error: sessionError } = await supabase.auth.exchangeCodeForSession(result.url);
        if (sessionError) {
          setError(sessionError.message);
          return;
        }
        // onAuthStateChange SIGNED_IN fires → setSession({ authStatus: 'registered' })
        router.replace('/(tabs)');
      } catch {
        setError('Google sign in failed. Please try again.');
      }
    };
    ```

    ── handleApple ───────────────────────────────────────────────────────────────

    Uses native Apple Sign In SDK. Note: creates a NEW Supabase session (different from
    the anonymous userId). The anonymous user's data remains in AsyncStorage — no migration
    needed for Phase 3 since data is device-local. Full data migration deferred to Phase 9.

    ```ts
    const handleApple = async () => {
      setError(null);
      try {
        // audit-added S3: nonce prevents Apple identity token replay attacks.
        // Raw nonce passed to signInWithIdToken; SHA-256 hash passed to signInAsync.
        const rawNonce = Math.random().toString(36).substring(2, 18);
        const hashedNonce = await Crypto.digestStringAsync(
          Crypto.CryptoDigestAlgorithm.SHA256,
          rawNonce
        );
        const credential = await AppleAuthentication.signInAsync({
          requestedScopes: [
            AppleAuthentication.AppleAuthenticationScope.FULL_NAME,
            AppleAuthentication.AppleAuthenticationScope.EMAIL,
          ],
          nonce: hashedNonce,
        });
        if (!credential.identityToken) {
          setError('Apple Sign In did not return an identity token');
          return;
        }
        const { error: authError } = await supabase.auth.signInWithIdToken({
          provider: 'apple',
          token: credential.identityToken,
          nonce: rawNonce,
        });
        if (authError) {
          setError(authError.message);
          return;
        }
        // onAuthStateChange SIGNED_IN fires → setSession({ authStatus: 'registered' })
        router.replace('/(tabs)');
      } catch (err: unknown) {
        const appleError = err as { code?: string };
        if (appleError?.code === 'ERR_REQUEST_CANCELED') return; // User cancelled — stay on screen
        setError('Apple Sign In failed. Please try again.');
      }
    };
    ```

    ── Apple button: guard by isAvailableAsync result (audit-applied S1) ───────────

    Use `appleAvailable` state (set by isAvailableAsync useEffect) — not a raw Platform
    check. `Platform.OS === 'ios'` alone misses iOS < 13 and enterprise-managed devices.

    ```tsx
    {appleAvailable && (
    ```

    Avoid: using Alert.alert for auth errors — inline error text is better UX for forms.
