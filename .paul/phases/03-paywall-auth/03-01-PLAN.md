<objective>
## Goal
Install RevenueCat SDK, build a PaywallScreen with 7-day trial offer, and wire a paywall gate that shows once after onboarding and never again.

## Purpose
Phase 2 ends at notification-permission, dropping the user directly into tabs. Phase 3 inserts the monetisation gate between onboarding and the main app. The paywall gate is the single revenue touchpoint before the user reaches any feature — correct placement here determines the entire trial-to-subscription funnel.

## Output
- `src/lib/revenuecat.ts` — SDK initializer, graceful no-op when API keys absent
- `app/paywall.tsx` — full PaywallScreen: trial offer, purchase CTA, skip path
- `src/store/onboardingGateStore.ts` — `paywallSeen` flag added (persisted)
- `src/store/sessionStore.ts` — `setSubscriptionStatus` action added
- `app/(tabs)/_layout.tsx` — paywall gate check added after onboarding gate
- `app/_layout.tsx` — paywall screen registered in Stack + `initPurchases` called at startup
- `app.config.ts` — `react-native-purchases` plugin added

    3. Create src/lib/revenuecat.ts:
       ```ts
       import { Platform } from 'react-native';
       import Purchases from 'react-native-purchases';

       export function initPurchases(): void {
         const apiKey = Platform.select({
           ios: process.env.EXPO_PUBLIC_REVENUECAT_IOS_KEY,
           android: process.env.EXPO_PUBLIC_REVENUECAT_ANDROID_KEY,
           default: undefined,
         });

         if (!apiKey) {
           if (__DEV__) {
             console.warn(
               '[kibun:rc] RevenueCat API key not configured. ' +
               'Set EXPO_PUBLIC_REVENUECAT_IOS_KEY / EXPO_PUBLIC_REVENUECAT_ANDROID_KEY in .env.'
             );
           }
           return;
         }

         Purchases.configure({ apiKey });
       }
       ```
       - initPurchases is called once at app startup (Task 3 wires the call site in _layout.tsx)
       - Platform.select returns undefined on web — safe no-op
       - No throw on missing key — unlike supabase.ts, RevenueCat is non-blocking for core functionality
  </action>

    handlePurchase — restructured WITHOUT finally block (audit M1+M2+S1): <!-- audit-added -->
    ```ts
    const handlePurchase = async () => {
      if (purchasing) return;
      setPurchasing(true);
      try {
        const offerings = await Purchases.getOfferings();
        const pkg = offerings.current?.availablePackages[0];
        if (!pkg) {
          // S1: warn when no offerings — visible in Metro, silent in production
          if (__DEV__) {
            console.warn('[kibun:rc] No offerings found. Configure products in RevenueCat dashboard.');
          }
          // No pkg — proceed to tabs as a graceful dev fallback (not a purchase path)
        } else {
          const { customerInfo } = await Purchases.purchasePackage(pkg);
          // M2: Read actual entitlement — never hardcode 'trial'
          // 'premium' must match the entitlement identifier configured in RevenueCat dashboard
          const active = customerInfo.entitlements.active['premium'];
          if (active) {
            setSubscriptionStatus(active.periodType === 'TRIAL' ? 'trial' : 'active');
          }
        }
      } catch (error: unknown) {
        const rcError = error as { code?: string };
        if (rcError?.code === PURCHASES_ERROR_CODE.PURCHASE_CANCELLED_ERROR) {
          // M1: User cancelled native IAP dialog — stay on paywall, re-enable button
          // Do NOT set paywallSeen. Do NOT navigate. User chose to cancel.
          setPurchasing(false);
          return; // EXIT EARLY — skips setPaywallSeen + router.replace below
        }
        // All other errors: network, RC not configured, etc. — log and fall through
        if (__DEV__) {
          console.warn('[kibun:rc] Purchase failed (non-cancel):', error);
        }
      }
      // Reaches here only on: successful purchase, no offerings (dev), or non-cancel error
      setPaywallSeen();
      router.replace('/(tabs)');
    };
    ```

    Avoid: using `finally` for navigation — it fires even on cancellation (audit M1 root cause).
    Avoid: hardcoding 'trial' as subscriptionStatus — read from CustomerInfo.entitlements (audit M2).
