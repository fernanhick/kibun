## Goal
Initialize the kibun Expo project with TypeScript, configure Expo Router with a 4-tab navigation shell, and establish the folder structure, path aliases, and Zustand store foundation.

## Purpose
This is the skeleton everything else builds on. Phase 2 (onboarding) and all subsequent phases depend on navigation, folder conventions, and the global store being in place. Getting this right means every future plan starts from a clean, consistent base.

## Output
- Runnable Expo app (TypeScript, Expo Router) with 4 placeholder tab screens
- Folder structure: src/components/, src/store/, src/types/, src/constants/, src/hooks/
- Zustand store initialized (empty, ready for state slices)
- Path aliases configured (@components, @store, @types, @constants, @hooks)
- app.config.ts with kibun app metadata (name, slug, bundle IDs, icons placeholders)
</objective>

  <name>Task 1: Initialize Expo project with TypeScript and configure app.config.ts</name>
  <files>
    app.config.ts, package.json, tsconfig.json, babel.config.js
  </files>
  <action>
    1. Initialize with: `npx create-expo-app kibun --template expo-template-blank-typescript`
    2. Replace app.json with app.config.ts containing:
       - name: "kibun", slug: "kibun"
       - ios.bundleIdentifier: "com.kibun.app"
       - android.package: "com.kibun.app"
       - plugins: ["expo-router"]
       - scheme: "kibun" (required for Expo Router deep linking)
       - icon: "./assets/icon.png" (placeholder — will be replaced in Phase 9)
       - splash.image: "./assets/splash.png" (placeholder)
    3. Install Expo Router: `npx expo install expo-router expo-linking expo-constants expo-status-bar react-native-safe-area-context react-native-screens`
    4. Install Zustand and AsyncStorage: `npx expo install zustand @react-native-async-storage/async-storage` <!-- audit-added: AsyncStorage is a required peer dep for Zustand persist on RN -->
    5. Install babel-plugin-module-resolver for path aliases: `npm install --save-dev babel-plugin-module-resolver`
    6. Configure tsconfig.json with strict mode and path aliases:
       - `"@components/*": ["src/components/*"]`
       - `"@store/*": ["src/store/*"]`
       - `"@types/*": ["src/types/*"]`
       - `"@constants/*": ["src/constants/*"]`
       - `"@hooks/*": ["src/hooks/*"]`
    7. Configure babel.config.js with module-resolver plugin matching tsconfig aliases
    <!-- audit-added: environment variable pattern -->
    8. Create .env.example with documented placeholder keys:
       - EXPO_PUBLIC_SUPABASE_URL=your-supabase-url (client-safe, used in Plan 01-02)
       - EXPO_PUBLIC_SUPABASE_ANON_KEY=your-supabase-anon-key (client-safe)
       - Note: OPENAI_API_KEY goes server-side only (Supabase Edge Function) — never in client env
    9. Verify .gitignore excludes: .env, .env.local, .env.*.local, node_modules/, .expo/
    Avoid: Using expo-router v2 API patterns if SDK is 52+ (use v3/v4 file-based routing)
    Avoid: Adding unnecessary packages — only what this plan requires
    Avoid: Committing any .env file — .env.example only
  </action>
  <verify>`npx tsc --noEmit` passes with zero errors; `npx expo-doctor` reports no critical issues; .env.example exists; `git check-ignore .env` confirms .env is gitignored</verify>

       - Set initialRouteName to "(tabs)"
    2. Create app/(tabs)/_layout.tsx as tab layout:
       - Use `&lt;Tabs&gt;` from expo-router
       - 4 tabs: Home (index), History, Insights, Settings
       - Tab bar icons: use @expo/vector-icons (Ionicons) — home-outline, time-outline, bar-chart-outline, settings-outline
       - Tab bar style: white background, active tint using primary color token (#6C63FF placeholder — will be replaced in Plan 01-03)
    3. Create each tab screen as a minimal placeholder:
       - Each file: imports Screen wrapper, renders a centered `&lt;Text&gt;` with screen name
       - Keep screens stateless and empty — data wiring happens in later phases
    <!-- audit-added: WCAG 2.1 AA accessibility labels — stated quality gate in PROJECT.md -->
    4. On each `<Tabs.Screen>` in the tab layout, set:
       - `tabBarAccessibilityLabel`: "Home tab", "History tab", "Insights tab", "Settings tab"
       - `tabBarButton` does NOT suppress the default accessible behavior
    <!-- audit-added: error boundary -->
    5. Add a minimal ErrorBoundary class component in app/_layout.tsx:
       - Catches unhandled render errors in the navigation tree
       - Renders a safe fallback view with a "Restart app" CTA
       - Logs the error message to console.error (Sentry integration deferred to Phase 9)
    Avoid: Implementing any screen logic or data fetching — placeholder UI only
    Avoid: Hardcoding colors — use a placeholder constant that Plan 01-03 will replace
  </action>
