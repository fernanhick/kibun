<task type="auto">
  <name>Task 1: Cold-start notification routing fix</name>
  <files>app/_layout.tsx</files>
  <action>
    Modify `app/_layout.tsx` to buffer notification navigation until the app is fully ready.

    **Step 1 — Update React import to add `useRef`:**
    ```typescript
    // Before:
    import React, { useState, useEffect } from 'react';
    // After:
    import React, { useState, useEffect, useRef } from 'react';
    ```

    **Step 2 — Update expo-router import to add `type Href`:**
    ```typescript
    // Before:
    import { Stack, useRouter } from 'expo-router';
    // After:
    import { Stack, useRouter, type Href } from 'expo-router';
    ```

    **Step 3 — Add readiness refs and pending route ref inside `RootLayout`, immediately after the existing state declarations:**
    ```typescript
    export default function RootLayout() {
      const router = useRouter();
      const { isReady } = useAuth();
      const [splashDone, setSplashDone] = useState(false);

      // Refs for cold-start notification routing.
      // isReadyRef and splashDoneRef give the notification listener stable access
      // to readiness state without adding them as dependencies (avoids re-registering
      // the listener on every render tick during startup).
      const isReadyRef = useRef(false);
      const splashDoneRef = useRef(false);
      const pendingRouteRef = useRef<string | null>(null);

      // Keep refs in sync with state
      useEffect(() => { isReadyRef.current = isReady; }, [isReady]);
      useEffect(() => { splashDoneRef.current = splashDone; }, [splashDone]);

      // Consume buffered cold-start navigation once both auth and splash have resolved.
      useEffect(() => {
        if (isReady && splashDone && pendingRouteRef.current) {
          router.push(pendingRouteRef.current as Href);
          pendingRouteRef.current = null;
        }
      }, [isReady, splashDone, router]);
    ```

    **Step 4 — Replace the existing notification deep-link useEffect with the guarded version:**
    ```typescript
    // Before:
    useEffect(() => {
      const sub = Notifications.addNotificationResponseReceivedListener((response) => {
        const data = response.notification.request.content.data as Record<string, unknown>;
        if (data?.type === 'ai_report') {
          router.push('/ai-report');
        }
      });
      return () => sub.remove();
    }, [router]);

    // After:
    // Route push notification taps to the appropriate screen.
    // Only ai_report type is currently handled — other types fall through silently.
    // On cold start (app killed → user taps push), the listener may fire before the
    // Stack is mounted. Buffer the route in pendingRouteRef; the consume effect above
    // will execute it once isReady && splashDone are both true.
    useEffect(() => {
      const sub = Notifications.addNotificationResponseReceivedListener((response) => {
        const data = response.notification.request.content.data as Record<string, unknown>;
        if (data?.type === 'ai_report') {
          if (!isReadyRef.current || !splashDoneRef.current) {
            // Cold-start: navigation tree not yet mounted — buffer for post-ready execution.
            pendingRouteRef.current = '/ai-report';
          } else {
            router.push('/ai-report' as Href);
          }
        }
      });
      return () => sub.remove();
    }, [router]);
    ```

    Key constraints:
    - `isReadyRef` and `splashDoneRef` are updated via their own `useEffect` hooks (not inline) to avoid the lint rule "don't mutate refs during render"
    - The listener `useEffect` depends only on `[router]` — registered once, stable. Readiness is read from refs not closure over state
    - `pendingRouteRef.current = null` after consuming prevents double-navigation on re-renders
    - `as Href` cast required (same pattern as '`/account' as Href` in settings.tsx — typed routes don't include stack-only screens until prebuild)
    - The three new `useEffect` hooks are inserted BEFORE the existing `useEffect` for reschedule-notifications
  </action>
  <verify>
    Run `npx tsc --noEmit`. Confirm:
    - No TS2304 for `useRef`
    - No TS2304 for `Href`
    - No TS2345 for `router.push(pendingRouteRef.current as Href)`
    - No lint errors on ref mutation (mutations are inside `useEffect`, not during render)
  </verify>
  <done>Cold-start notification routing fix complete. Listener buffers /ai-report navigation until isReady && splashDone. Pending route consumed exactly once.</done>

## Stale Subscription Note (QG-3)
`sessionStore.subscriptionStatus` is set at purchase time and not refreshed against RevenueCat on launch. An expired subscriber retains 'trial' or 'active' in the store indefinitely. This does not cause a crash (QG-3 passes), but premium features remain accessible post-expiry until the user signs out. Fix in v1.1: call `Purchases.getCustomerInfo()` on app launch (after `isReady`) and sync `subscriptionStatus` from `entitlements.active`.
