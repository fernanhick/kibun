<objective>
## Goal
Install and configure the Supabase client with expo-secure-store for JWT token storage, implement anonymous sign-in on first launch, and wire auth state into the existing Zustand session store.

## Purpose
Every subsequent phase depends on a user identity — mood entries, profile data, and AI reports all require a userId. This plan establishes the auth layer so Plans 01-03 onward build on a real, persistent identity rather than placeholder state. Anonymous auth means zero friction: the user has a real Supabase user ID from the first second of app launch.

## Output
- `src/lib/supabase.ts` — Supabase client singleton with expo-secure-store token adapter
- `src/hooks/useAuth.ts` — hook that initializes anonymous auth and subscribes to auth state changes
- `app/_layout.tsx` updated — calls useAuth on mount, gates app render until session is ready
- Zustand sessionStore populated with real userId, authStatus, subscriptionStatus from Supabase
</objective>

  <action>
    1. Install dependencies:
       `npx expo install @supabase/supabase-js expo-secure-store`
    2. Create `src/lib/supabase.ts` as the Supabase client singleton:
       - Import `createClient` from `@supabase/supabase-js`
       - Import `* as SecureStore` from `expo-secure-store`
       - Create `ExpoSecureStoreAdapter` implementing the Supabase storage interface:
         ```
         getItem: (key) => SecureStore.getItemAsync(key)
         setItem: (key, value) => SecureStore.setItemAsync(key, value)
         removeItem: (key) => SecureStore.deleteItemAsync(key)
         ```
       - Read URL and key from `process.env.EXPO_PUBLIC_SUPABASE_URL` and `process.env.EXPO_PUBLIC_SUPABASE_ANON_KEY`
       - Throw a clear error at module load if either env var is missing (fail fast, not silently)
       - Initialize client with:
         ```
         auth: {
           storage: ExpoSecureStoreAdapter,
           autoRefreshToken: true,
           persistSession: true,
           detectSessionInUrl: false,   ← required for React Native (no URL detection)
         }
         ```
       - Export as named export: `export const supabase`
    Avoid: Using AsyncStorage for auth tokens — SecureStore is required (per 01-01 security contract)
    Avoid: Hardcoding Supabase URL or key — env vars only
    Avoid: Default export — named export keeps imports explicit
  </action>
  <verify>`npx tsc --noEmit` passes with zero errors; src/lib/supabase.ts exists and exports `supabase`</verify>
  <done>AC-1 satisfied: Client compiles clean with correct config</done>

<task type="auto">
  <name>Task 2: Create useAuth hook and wire into root layout</name>
  <files>
    src/hooks/useAuth.ts,
    app/_layout.tsx
  </files>
  <action>
    1. Create `src/hooks/useAuth.ts`:
       - Import `supabase` from `@lib/supabase` (add @lib to path aliases — see note below)
       - Import `useSessionStore` from `@store/index`
       - Implement `useAuth()` hook with local state: `isReady: boolean` (default false)
       - On mount (useEffect with [] deps):
         a. Subscribe to auth state changes via `supabase.auth.onAuthStateChange`:
            - On INITIAL_SESSION (fires automatically on subscribe — Supabase v2):
              - If `session` is not null: call `setSession(...)`, set `isReady = true`
              - If `session` is null (no persisted session): call `supabase.auth.signInAnonymously()`
                - On error: log structured error `console.error('[kibun:auth]', { event: 'anon_sign_in_failed', error })`, set `isReady = true` anyway (app must not hang)
                - On success: SIGNED_IN will fire and handle store population
            - On SIGNED_IN: call `setSession({ userId: session.user.id, authStatus: session.user.is_anonymous ? 'anonymous' : 'registered', subscriptionStatus: 'none' })`, set `isReady = true`
            - On TOKEN_REFRESHED: call `setSession(...)` with updated session data
            - On SIGNED_OUT: call `clearSession()`
         Note: Do NOT call `supabase.auth.getSession()` separately — INITIAL_SESSION replaces it and avoids race conditions between the listener and imperative checks.
       - Cleanup on unmount: call `subscription.unsubscribe()`
       - Return `{ isReady }`
    2. Add `@lib` path alias — update tsconfig.json and babel.config.js:
       - tsconfig: `"@lib/*": ["src/lib/*"]`
       - babel: `'@lib': './src/lib'`
    3. Update `app/_layout.tsx`:
       - At module level (outside component): call `SplashScreen.preventAutoHideAsync()` from `expo-splash-screen` — keeps native splash visible during auth init
       - Import and call `useAuth()` at the top of `RootLayout`
       - Use `useEffect` dependent on `isReady`: when `isReady` becomes true, call `SplashScreen.hideAsync()`
       - If `!isReady`, return `null` (belt-and-suspenders: splash screen covers this but null prevents any partial render leak)
       - Once `isReady` is true, render the full Stack navigator as before
    Avoid: Calling `supabase.auth.getSession()` — INITIAL_SESSION event handles session restoration without the race condition
    Avoid: Blocking the app indefinitely — always set isReady=true even on auth error
    Avoid: Storing auth tokens in sessionStore — only metadata (userId, authStatus, subscriptionStatus)
    Avoid: Skipping `SplashScreen.preventAutoHideAsync()` — without it the native splash auto-dismisses before auth resolves, causing a blank screen flash
  </action>
  <verify>
    - `npx tsc --noEmit` passes with zero errors
    - `src/hooks/useAuth.ts` exists and exports `useAuth`
    - `app/_layout.tsx` imports and calls `useAuth`, returns null when !isReady
    - `@lib/supabase` import resolves in TypeScript
  </verify>
  <done>AC-2, AC-3, AC-4, AC-5 satisfied: anonymous auth initializes, session persists, store synced, app gates on isReady</done>
